# Writing Rules

These rules apply to every bullet, every line, every word. Read this before writing anything.

---

## The single biggest AI tell: em dashes

Never use em dashes (—) anywhere in the resume.

Em dashes are the most flagged AI pattern in 2025-2026. Enterprise ATS systems (Workday, Greenhouse, Lever) shipped content classifiers that downgrade resumes matching GPT defaults. Em dash density is one of the top signals.

Replace em dashes with:
- A comma
- A colon
- A period (split into two sentences)
- The word "and," "so," or "which"
- Parentheses for asides

**Wrong:** Built a pipeline that scrapes DOL API — filters non-companies — outputs to Google Sheets
**Right:** Built a pipeline that scrapes DOL API, filters non-companies, and outputs to Google Sheets

## Verbs to never use

These read as AI-generated. Replace them with concrete alternatives.

| Banned | Use instead |
|---|---|
| Leveraged | used, applied, built with |
| Spearheaded | led, started, launched |
| Demonstrated | showed, proved, delivered |
| Drove | grew, pushed, increased |
| Utilized | used |
| Optimized | improved, cut, reduced, raised |
| Facilitated | ran, managed, helped |
| Streamlined | simplified, cut, rebuilt |
| Orchestrated | ran, managed, coordinated |
| Championed | pushed for, advocated |
| Collaborated | worked with |
| Interfaced | talked to, worked with |

## Phrases to never use

- "Results-driven professional"
- "Detail-oriented"
- "Team player"
- "Strong communication skills" (show it, do not claim it)
- "Passionate about"
- "Proven track record" (say what the track record actually is)
- "Dynamic environment"
- "Fast-paced"
- "Mission-critical"
- "Scalable solutions"
- "Innovative approaches"
- "Value-add"

---

## What good bullets look like

**Bad (generic, no decisions, AI-sounding):**
"Leveraged advanced AI technologies to develop scalable, mission-critical solutions that drove significant business value across diverse client engagements."

**Good (specific, concrete, shows judgment):**
"Built and shipped 50 AI systems across 30 clients: voice agents, inbound receptionists, SMS bots, and cold email tools. Owned the full lifecycle from scoping to post-deployment support with no oversight."

---

## The decisions test

For project bullets especially: if you could swap this bullet onto anyone else's resume, it is not specific enough.

Good bullets include:
- A specific decision and why it was made
- A number (50 systems, 30 clients, 14 stages, 49% accuracy)
- A named tool or API
- An outcome (a shorter correct list beats a longer dirty one)

## Honest skills framing

The skills section is the most commonly lied-about section. If a recruiter or interviewer probes any skill and the candidate cannot demonstrate it, the whole resume loses credibility.

**Rule:** Only list skills the person can demonstrate in a live setting without preparation.

| Situation | How to write it |
|---|---|
| Knows the language, writes it independently | "Python" or "Python (data pipelines)" |
| Can read and understand but not write independently | "Familiar with Python" |
| Built with it via AI coding tool (Claude Code, Cursor, etc.) | List the AI tool, not the language |
| Used a platform's UI without writing code | List the platform: "Supabase," "Apollo," "n8n" |
| Never used it | Do not list it |

Never list a skill just because the user wants it on there. The interview will expose it.

## Summary paragraph rules

- Maximum 4 sentences
- First sentence: who they are + strongest signal
- Middle: what they have actually built or done
- Last: something specific about mindset or working style
- No abstract adjectives
- No em dashes
- Write it last, after all bullets are done

## General writing rules

**Specific beats vague.** "50+ AI systems across 30+ clients" beats "numerous client engagements."

**Decisions beat actions.** "Chose ChromaDB over Postgres because the use case required semantic search" beats "used ChromaDB for the vector database."

**Outcomes beat duties.** "Raised domain resolution accuracy from 49% to near-certain" beats "improved domain resolution."

**Short sentences beat long ones.** If a bullet runs longer than two lines, split it.

**Parallel structure.** All bullets in a section start with the same grammatical form (verb, noun, or noun phrase).

**Do not start consecutive bullets with the same word.** Vary the opening verb.

## Deeper AI tells: beyond em dashes and banned verbs

Em dashes and banned verbs are the obvious tells. Enterprise ATS content classifiers (and any recruiter who's read 200 GPT-written resumes this year) also catch these subtler patterns:

**Sentence-length uniformity.** Human writing has burstiness: short bullets next to long ones, a fragment next to a full sentence. AI output defaults to a narrow band, most bullets landing within a few words of the same length. Check the bullet list for a whole section: if every bullet is within 3-4 words of the same length, vary it. Cut one down to a single clause, let another run longer if it earns it.

**Too-perfect parallel structure.** Parallel structure (see above) is correct in principle, but AI writing takes it further than a human would: every single bullet in a section starting with [Verb] + [Object] + [Tool] + [Metric], in that exact slot order, every time. A human writer breaks the pattern occasionally, leading with the metric once, or the tool, or dropping the metric entirely when there isn't a clean one. If every bullet in a section is structurally identical down to the clause order, restructure at least one.

**Rule of three overuse.** "Scoping, building, and deploying." "Fast, reliable, and scalable." AI defaults to listing exactly three items grouped in a triplet, especially in summaries and skill framing. Two items or four read more human. If a resume has more than one rule-of-three list, break at least one.

**Symmetric sentence construction.** "Not only did X, but also Y" or "By doing X, achieved Y" used repeatedly across bullets. One use is fine. Repetition across a section is a tell.

Check the full draft against these four before calling it done, the same way the copy-paste test below checks tone.

---

## The copy-paste test for AI tells

Before finalising, read every bullet aloud. If it sounds like a product brochure or a press release, rewrite it. It should sound like an engineer or professional explaining their work to a colleague over coffee.
