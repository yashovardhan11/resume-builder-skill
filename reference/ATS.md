# ATS Formatting Rules

All formatting decisions follow these rules. Do not deviate without a specific reason.

---

## Font

**Font family:** Arial throughout. No mixing fonts.

**Sizes (in docx half-points — multiply pt by 2):**

| Element | Point size | Half-point (docx size field) |
|---|---|---|
| Name | 20pt | 40 |
| Section headings | 13pt | 26 |
| Job/project title | 11pt | 22 |
| Date/meta/company | 10pt | 20 |
| Body text and bullets | 11pt | 22 |
| Contact line | 10pt | 20 |
| Education institution | 10pt | 20 |

**Default body size is 11pt, not 10.5pt.** Freshers rarely have enough content to need smaller text — reserve 10.5pt/10pt for genuine overflow, not as the starting point. See the fit-to-one-page ladder in `DOCX-TEMPLATE.md`.

**Minimum acceptable body size:** 10pt (size 20). Do not go below this even when trying to fit to one page. ATS parsers treat text below 10pt as noise.

**Name is always the largest element.** Section headings are the second largest. Everything else is smaller than both.

---

## Margins

| Setting | Value (DXA) | Inches |
|---|---|---|
| Standard | 1080 DXA | 0.75" |
| Tight (when needed to fit page) | 864 DXA | 0.6" |
| Minimum | 720 DXA | 0.5" |

Use standard margins by default. Move to tight only if content does not fit. Never go below minimum.

---

## Line spacing

| Use | Spacing | DXA value |
|---|---|---|
| Body text and bullets | 1.1x | 264 |
| Summary paragraph | 1.1x | 264 |
| Structural elements (headings, headers) | 1.0x | 240 |

Apply `LineRuleType.AUTO` consistently.

---

## Colours

| Role | Hex | Use |
|---|---|---|
| Accent | #1B3A6B | Section heading text, header underline border |
| Dark | #1A1A1A | All body text, name, job titles |
| Gray | #555555 | Dates, institutions, meta text, contact line |

---

## Section headings

- All caps
- Bold
- Accent colour
- Underline via `BorderStyle.SINGLE` bottom border on the paragraph (size 4, accent colour, space 3)
- Spacing: 120 before, 50 after

---

## Bullet points

- Use en dash (–) as the bullet character, not a hyphen or bullet dot
- In docx: `LevelFormat.BULLET` with `text: "\u2013"`
- Indent: `left: 400, hanging: 200`
- Spacing: 14-18 before and after each bullet

---

## Page setup

- Page size: US Letter (width 12240 DXA, height 15840 DXA)
- Single column only — no tables, no text boxes, no multi-column layouts
- No images, logos, or graphics
- No headers or footers (ATS parsers often miss content in header/footer regions)
- All content in the main document body

---

## Section header naming

Use standard names that ATS systems recognise:
- SUMMARY (not "About Me" or "Profile")
- EXPERIENCE (not "Work History")
- PROJECTS (not "Portfolio" or "Work Samples")
- SKILLS (not "Competencies" or "Expertise")
- EDUCATION (not "Academic Background")
- CERTIFICATIONS (if applicable)

---

## What kills ATS parsing

Never use any of these:
- Tables (ATS parsers scramble table content)
- Text boxes (content is often skipped entirely)
- Multi-column layouts
- Images or logos
- Headers and footers for contact info
- Fancy bullet symbols (use en dash only)
- Non-standard fonts
- Colours for body text (accent for headings only)
- Hyperlinks with custom anchor text (spell out URLs plainly)

---

## File format

Output is both `.docx` and `.pdf` — the PDF is generated by converting the validated `.docx` (see `DOCX-TEMPLATE.md` — "PDF conversion"), never created independently. Run the copy-paste test on the PDF before delivering: select all text, copy, paste into a text editor. If it copies cleanly, the ATS can read it.

---

## Validation

Always run after generating:
```bash
python3 ~/.claude/skills/docx/scripts/office/validate.py <output_path>
```

Fix any errors before delivering. Do not present an unvalidated file.
